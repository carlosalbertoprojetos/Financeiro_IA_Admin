# Feature Adoption Tracker

Data: 2026-06-27

## Objetivo

Medir quais partes do EOR geram valor percebido, confianca e uso recorrente durante validacoes com usuarios reais.

## Escala

Frequencia de uso:

- 0: nao usado.
- 1: usado apenas quando guiado.
- 2: usado espontaneamente uma vez.
- 3: usado espontaneamente mais de uma vez.
- 4: usado como parte central da decisao.

Utilidade percebida:

- 1: sem valor.
- 2: valor baixo.
- 3: util em alguns casos.
- 4: util para decisao.
- 5: essencial para decisao.

Confianca:

- 1: nao confiavel.
- 2: muitas duvidas.
- 3: confiavel com ressalvas.
- 4: confiavel.
- 5: altamente confiavel.

## Sinais De Adocao Forte

- Usuario volta espontaneamente para a feature.
- Usuario usa a feature para justificar uma decisao.
- Usuario compartilha ou pede export.
- Usuario pergunta como usar em rotina.
- Usuario compara com processo atual e percebe economia de tempo.

## Sinais De Baixa Adocao

- Usuario ignora a feature.
- Usuario so entende apos explicacao longa.
- Usuario nao consegue relacionar a feature a uma decisao.
- Usuario questiona confianca sem encontrar evidencia.
- Usuario considera a informacao redundante.
