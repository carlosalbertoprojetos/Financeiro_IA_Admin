# Customer Insights Backlog

Data: 2026-06-27

## Objetivo

Transformar evidencias reais de validacao com clientes em backlog priorizado do EOR.

Este backlog nao deve conter ideias internas sem evidencia. Cada item precisa apontar para observacao, comentario, decisao, metrica ou objecao coletada durante o Customer Validation Program.

## Criterios De Priorizacao

P0 - Obrigatorio antes de ampliar piloto ou vender:

- Bloqueia confianca.
- Bloqueia uso recorrente.
- Impede decisao executiva.
- Gera risco operacional ou comercial alto.
- Foi observado em usuario real com severidade alta ou bloqueadora.

P1 - Melhora forte de valor:

- Aumenta clareza, utilidade ou confianca.
- Reduz tempo de analise.
- Aumenta adocao de feature importante.
- Resolve dor recorrente, mas nao bloqueia piloto.

P2 - Refinamento:

- Melhora apresentacao, detalhe ou conveniencia.
- Tem valor potencial, mas evidencia ainda e limitada.
- Pode aguardar nova rodada de validacao.

## Regra De Entrada

Para entrar no backlog, o item deve ter:

- Evidencia ID.
- Tipo de evidencia.
- Usuario ou perfil afetado.
- Impacto esperado.
- Severidade.
- Relacao clara com valor, confianca, decisao ou adocao.

## Regra De Saida

Um item pode ser removido ou rebaixado se:

- A evidencia nao se repetir.
- O impacto for baixo.
- A necessidade for resolvida por orientacao ou onboarding.
- O pedido for uma preferencia individual sem dor operacional.
